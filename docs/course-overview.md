# Course Overview

## What This Course Teaches

This course teaches Python in a way that directly supports Data Engineering work. Instead of treating Python as a disconnected programming subject, it introduces language features through file ingestion, record processing, validation, databases, APIs, and pipeline design.

## Core Outcomes

By the end of this repository, a learner should be able to:

- write readable Python code for data tasks
- process records from CSV, JSON, and text files
- clean and validate raw inputs
- build reusable transformation functions
- load data into a relational database
- extract data from APIs responsibly
- structure simple ETL pipelines
- add logging, testing, and configuration
- explain tradeoffs around performance and scale

## Why Python Matters in Data Engineering

Python is not the only language used in Data Engineering, but it is one of the most useful because it is:

- easy to read and teach
- strong for automation
- strong for scripting and orchestration
- rich in libraries for files, APIs, and databases
- widely used in analytics, data science, and platform tooling

Python often becomes the glue between storage systems, APIs, schedulers, and transformation logic.

## How The Repo Is Organized

- `docs/` explains how to navigate, study, and set up the course
- `modules/` contains the core lessons
- `exercises/` contains practice questions and separate solutions
- `datasets/` contains small mock data for exercises and projects
- `src/` contains starter Python utilities and pipeline code
- `projects/` contains mini-projects and capstones
- `tests/` contains unit and integration tests

## Teaching Approach

Each module includes:

- learning objectives
- concept explanations
- why the topic matters in Data Engineering
- hands-on examples
- exercises and challenge tasks
- common mistakes
- a summary and next-step bridge

## Recommended Study Method

1. Read the lesson markdown.
2. Run the sample Python code.
3. Complete the exercises without checking solutions.
4. Compare your answer to the solution.
5. Improve one script to make it more readable or reusable.
6. Keep a running list of Data Engineering use cases for each concept.

## Portfolio Advice

The strongest learners do not stop at completing the exercises. They:

- refactor one mini-project into cleaner modules
- add tests and validation
- document assumptions
- explain architecture choices
- publish the project with a clean README
