# Course Roadmap

This roadmap takes a learner from Python beginner to someone who can build small but realistic Data Engineering pipelines.

## Stage 1: Foundations

### Module 00. Orientation
- Understand what Data Engineering is
- Learn why Python is useful in the field
- Set up tools and the repo workflow

### Module 01. Python Basics
- Variables, types, operators, conditionals, loops, and functions
- Readability, debugging, and beginner-safe coding habits

### Module 02. Data Structures
- Strings, lists, tuples, sets, and dictionaries
- Slicing, iteration patterns, and built-in functions
- Working with records the way raw data often arrives

## Stage 2: Practical Python

### Module 03. Intermediate Python
- Comprehensions, imports, modules, args/kwargs, exceptions, and basic OOP
- Reusable code for data workflows

### Module 04. Files and I/O
- Text files, CSV, JSON, paths, validation, encodings, and compression
- Real ingestion patterns for local and shared files

### Module 05. Data Processing
- Cleaning, transforming, aggregating, schema validation, missing values, and type conversion
- Building reusable transformation functions

## Stage 3: Core Data Engineering Inputs and Outputs

### Module 06. Databases
- SQLite-based examples for SQL execution, parameterized inserts, transactions, and reads
- Loading transformed data into tables safely

### Module 07. APIs
- Requesting data, handling JSON, pagination, rate limits, and authentication basics
- Extract scripts that produce clean, load-ready data

### Module 08. ETL / ELT
- Pipeline stages, data layers, idempotency, metadata, and config-driven jobs
- Thinking in repeatable, operational workflows

## Stage 4: Production Skills

### Module 09. Production Python
- Logging, configuration, environment variables, CLI usage, testing, retries, and refactoring
- Turning scripts into maintainable code

### Module 10. Performance
- Time and memory basics
- Generators, iterators, chunking, and when Python is enough
- When to move to distributed tools like Spark

### Module 11. Project Structure
- Organize code, tests, docs, config, notebooks, and reusable utilities
- Separate exploratory work from production code

## Stage 5: Real-World Pipeline Thinking

### Module 12. Real-World Patterns
- Full refresh vs incremental loads
- Deduplication, late-arriving data, partitioning, audit columns, observability, reproducibility

### Module 13. Mini Projects
- Practice realistic single-purpose pipelines
- Build confidence before larger case studies

### Module 14. Capstones
- End-to-end pipeline projects with business context
- Architecture explanation, tasks, and solution guidance

## Stage 6: Career Readiness

### Module 15. Interview Prep
- Python interview questions for Data Engineering
- Debugging habits, common mistakes, anti-patterns, and maintainability rules

### Module 16. Next Steps
- SQL, data modeling, orchestration, Spark, Docker, cloud platforms, observability, and lakehouse concepts

## Recommended Path

- If you are new to programming: complete every module in order.
- If you already know Python basics: skim modules 01 and 02, then go deep from module 04 onward.
- If you already work with data: prioritize modules 04 to 12 and complete at least two projects.

## Completion Milestones

- Milestone 1: Write readable Python functions and process basic records
- Milestone 2: Build scripts that ingest files and JSON data
- Milestone 3: Load clean data into a database with validation and logging
- Milestone 4: Structure a small ETL project like a real repository
- Milestone 5: Ship a capstone project that demonstrates Data Engineering thinking
