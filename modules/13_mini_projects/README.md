# Module 13: Practical Mini Projects

## Learning Objectives

- Build several small but realistic data pipelines
- Practice extracting, validating, transforming, and loading
- Strengthen project documentation habits

## Included Mini Projects

- CSV sales ETL pipeline
- JSON API ingestion pipeline
- log file parser
- data validation pipeline
- SQLite loader
- batch file processing pipeline
- daily reporting script
- config-driven ingestion script

## Why It Matters in Data Engineering

Mini-projects are where isolated lessons start to feel like real work. They help you combine files, code structure, validation, logging, and outputs in a small but complete workflow.

## Suggested Approach

1. Read the project brief
2. Inspect the input data
3. Implement the starter tasks
4. Add one improvement of your own
5. Document the architecture in the project README

## Common Mistakes

- Jumping into code without understanding the data flow
- Skipping validation and logging
- Keeping all logic in one script
- Forgetting to describe assumptions

## Summary

Mini-projects help you convert learning into evidence of skill.
