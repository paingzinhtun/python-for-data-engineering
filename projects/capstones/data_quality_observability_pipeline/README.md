# Capstone: Data Quality and Observability Pipeline

## Project Overview

Build a pipeline focused on validation, rejection handling, audit columns, and run reporting rather than just happy-path loading.

## Business Context

A team trusts source data too much and repeatedly ships broken dashboards. Your goal is to improve confidence in the pipeline by adding strong checks and visibility.

## Tasks

1. Define quality checks for required fields, duplicates, and numeric ranges.
2. Split valid and invalid rows.
3. Write an audit summary with counts and reasons.
4. Add tests for each quality rule.
5. Explain how the design supports debugging and reproducibility.

## Expected Learning Outcomes

- quality-first pipeline thinking
- observability basics
- maintainable validation design
- business-aware debugging
